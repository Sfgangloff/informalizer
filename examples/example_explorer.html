11/-!
12# Symbolic dynamics on cancellative monoids
13 
14This file develops a minimal API for symbolic dynamics over a
15**left-cancellative monoid** `G`—formally, a structure carrying `[Monoid G]`
16and `[IsLeftCancelMul G]` (which becomes `[AddMonoid G]` and
17`[IsLeftCancelAdd G]` in the additive form). Throughout the documentation we use the
18**additive** notations, which are the most common in symbolic dynamics, although
19all the notions introduced are defined in the multiplicative notations and adapted
20to the additive notation.
21 
22Given a finite alphabet `A`, the ambient configuration space is the set of
23functions `G → A`, endowed with the product topology. We define the
24left-translation action, cylinders, finite patterns, their occurrences,
25forbidden sets, and subshifts (closed, shift-invariant subsets). Basic
26topological facts (e.g. cylinders are clopen, occurrence sets are clopen,
27forbidden sets are closed) are proved under discreteness assumptions on
28the alphabet.
29 
30The development is generic for left-cancellative monoids. This covers both
31groups (the standard setting of symbolic dynamics) and more general monoids
32where cancellation holds but inverses may not exist. Geometry specific to
33`ℤ^d` (boxes/cubes and the box-based entropy) is deferred to a separate
34specialization.
35 
36## Why cancellativity?
37 
38Some constructions, such as translating a finite pattern to occur at a point `v`,
39require solving equations of the form `w + v = h`. For this to have a unique
40solution `w` given `h` and `v`, we assume **left-cancellation**:
41if `v + a = v + b` then `a = b`. This allows us to define
42`Pattern.shift` (which shifts a pattern) without using inverses,
43so that the theory works not only for groups but also for cancellative monoids.
44 
45## Main definitions
46 
47* `shift g x` — left translation: in additive notation `(shift v x) u = x (v + u)` (using the
48**left** action of `G` on configurations).
49* `cylinder U x` — configurations agreeing with `x` on a finite set `U ⊆ G`.
50* `Pattern A G` — a configuration which takes
51default value outside of a finite support, together with this support.
52* `Pattern.occursInAt p x g` — occurrence of `p` in `x` at translate `g`.
53* `forbidden F` — configurations avoiding every pattern in `F`.
54* `Subshift A G` — closed, shift-invariant subsets of the full shift.
55* `MulSubshift.ofForbidden F` — the subshift defined by forbidding a family of patterns.
56* `subshift_of_finite_type F` — a subshift of finite type defined by a finite set of
57forbidden patterns.
58* `languageOn X U` — the set of patterns of shape `U` obtained by restricting some `x ∈ X`.
59 
60## Design choice: ambient vs. inner (subshift-relative) viewpoint
61 
62All core notions (shift, cylinder, occurrence, language, …) are defined **in the
63ambient full shift** `G → A`. A subshift is then a closed, invariant subset,
64bundled as `Subshift A G`. Working inside a subshift is done by restriction.
65 
66**Motivation.**
67 
68If cylinders and shifts were defined only *inside* a subshift, local ergonomics
69would improve but global operations would become awkward. For instance, to prove
70that for finite shape `U`:
71 
72`languageOn (X ∪ Y) U = languageOn X U ∪ languageOn Y U,`
73 
74one must eventually move both sides to the ambient pattern type. Similar issues
75arise for intersections, factors, and products. By contrast, with ambient
76definitions these set-theoretic identities are tautological.
77Thus the file develops the theory ambiently, and subshifts reuse it by restriction.
78 
79**Working inside a subshift.**
80 
81For `Y : Subshift A G`, cylinders and occurrence sets *inside `Y`* are simply
82preimages of the ambient ones under the inclusion `Y → (G → A)`. For example:
83 
84`{ y : Y | ∀ i ∈ U, (y : G → A) i = (x : G → A) i } = (Subtype.val) ⁻¹' (cylinder U (x : G → A)).`
85 
86Shift invariance guarantees that the ambient shift restricts to `Y`.
87 
88**Ergonomics.**
89 
90Thin wrappers (e.g. `Subshift.shift`, `Subshift.cylinder`, `Subshift.languageOn`)
91may be added for convenience. They introduce no new theory and unfold to the
92ambient definitions.
93 
94## Namespacing policy
95 
96All ambient definitions live under the namespace `SymbolicDynamics.FullShift`.
97If inner, subshift-relative wrappers are provided, they will be placed in the
98subnamespace `SymbolicDynamics.Subshift`. This separation avoids name clashes
99between the two viewpoints, since both may naturally want to reuse names like
100`cylinder`, `shift`, `occursAt`, or `languageOn`.

119section ShiftDefinition

121variable {A G : Type*} [Monoid G]
122 
123/-- The **left-translation shift** on configurations.
124 
125We call *configuration* an element of `G → A`.
126 
127Given a configuration `x : G → A` and an element `g : G` of the monoid, the shifted configuration
128`mulShift g x` is defined by `(mulShift g x) h = x (g * h)`.
129 
130Intuitively, this moves the whole configuration "in the direction of `g`": the value
131at position `h` in the shifted configuration is the value that was at position
132`g * h` in the original one.
133 
134For example, if `G = ℤ` (with addition) and `A = {0, 1}`, then
135`mulShift 1 x` is the sequence obtained from `x` by shifting every symbol one
136step to the left. -/
137@[to_additive /-- The **left-translation shift** on configurations, in additive notation.
138 
139We call *configuration* an element of `G → A`.
140 
141Given a configuration `x : G → A` and an element `g : G` of the additive monoid,
142the shifted configuration `shift g x` is defined by `(shift g x) h = x (g + h)`.
143 
144Intuitively, this moves the whole configuration "in the direction of `g`": the value
145at position `h` in the shifted configuration is the value that was at position
146`g + h` in the original one.
147 
148For example, if `G = ℤ` and `A = {0, 1}`, then
149`shift 1 x` is the sequence obtained from `x` by shifting every symbol one
150step to the left. -/]
151def mulShift (g : G) (x : G → A) : G → A :=
152 fun h => x (g * h)
153 
154@[to_additive (attr := simp)] lemma mulShift_apply (g : G) (x : G → A) (h : G) :
155 mulShift g x h = x (g * h) := rfl
156 
157@[to_additive (attr := simp)] lemma mulShift_one (x : G → A) : mulShift (1 : G) x = x := by
158 ext h; simp [mulShift]
159 
160/-- Composition of left-translation shifts corresponds to multiplication in the monoid `G`. -/
161@[to_additive] lemma mulShift_mul (g₁ g₂ : G) (x : G → A) :
162 mulShift (g₁ * g₂) x = mulShift g₂ (mulShift g₁ x) := by
163 ext h; simp [mulShift, mul_assoc]
164 
165variable [TopologicalSpace A]

167/-- The left-translation shift is continuous. -/
168@[to_additive (attr := fun_prop)] lemma continuous_mulShift (g : G) :
169 Continuous (mulShift (A := A) g) := by
170 -- coordinate projections are continuous; composition preserves continuity
171 unfold mulShift
172 fun_prop

178section Cylinders

180variable {A G : Type*}

182/-- A *cylinder set* is the set of all configurations that agree with a given
183reference configuration `x` on a fixed finite subset `U` of the index set `G`.
184 
185The set `U` is called the *support* of the cylinder.
186 
187Intuitively, cylinders specify the "letters" on finitely many coordinates, while
188leaving all other coordinates free. For example, in the full shift `{0, 1}^ℤ`,
189the cylinder determined by `U = {0, 1}` and `x 0 = 1, x 1 = 0` consists of all
190bi-infinite sequences of `0`s and `1`s whose entries on positions `0` and `1`
191respectively are `1` and `0`.
192 
193When `A` has the discrete topology, cylinder sets form a basis of clopen sets
194for the product topology on `G → A`. -/
195def cylinder (U : Finset G) (x : G → A) : Set (G → A) :=
196 { y | ∀ i ∈ U, y i = x i }
197 
198/-- A cylinder set on `U` is the `Set.pi` over `U` of the singletons `{x i}`,
199viewed as a subset of `G → A`. Equivalently, it is the preimage of that product
200of singletons in `U → A` under the restriction map `(G → A) → (U → A)`. -/
201lemma cylinder_eq_set_pi (U : Finset G) (x : G → A) :
202 cylinder U x = Set.pi (↑U : Set G) (fun i => ({x i} : Set A)) := by
203 ext y; simp [cylinder, Set.pi]
204 
205lemma mem_cylinder {U : Finset G} {x y : G → A} :
206 y ∈ cylinder U x ↔ ∀ i ∈ U, y i = x i := Iff.rfl
207 
208variable [TopologicalSpace A]
209 
210/-- Cylinders are open when `A` is discrete. -/
211lemma isOpen_cylinder [DiscreteTopology A] (U : Finset G) (x : G → A) :
212 IsOpen (cylinder U x) := by
213 simpa [cylinder_eq_set_pi U x] using isOpen_set_pi (U.finite_toSet) (by simp)
214 
215/-- Cylinders are closed when `A` is a T1 Space. -/
216lemma isClosed_cylinder [T1Space A] (U : Finset G) (x : G → A) :
217 IsClosed (cylinder U x) := by
218 simpa [cylinder_eq_set_pi U x] using isClosed_set_pi (by simp)

220end Cylinders

224/-- A *subshift* on an alphabet `A` is a closed, shift-invariant subset of `G → A`. Formally, it is
225composed of:
226* `carrier`: the underlying set of allowed configurations.
227* `isClosed`: the set is topologically closed in `A^G`.
228* `mapsTo`: the set is invariant under all left-translation shifts
229 `(shift g)`. -/
230structure Subshift (A : Type*) [TopologicalSpace A] (G : Type*) [AddMonoid G] where
231 /-- The underlying set of configurations (additive monoid version). -/
232 carrier : Set (G → A)
233 /-- Closedness of `carrier`. -/
234 isClosed : IsClosed carrier
235 /-- Shift invariance of `carrier` for the additive shift `shift`. -/
236 mapsTo : ∀ g : G, MapsTo (shift g) carrier carrier

263@[to_additive fullShift

274/-- A *pattern* is a finite configuration in the full shift `A^G`.
275 
276It consists of:
277* a full configuration `config : G → A` in the full shift;
278* a finite subset `support : Finset G` of coordinates, called the support of `p`;
279* a proof `condition` that outside `support`, `config` takes the default value of `A`.
280 
281Intuitively, a pattern is a "partial configuration" specifying finitely many values of
282a configuration in `G → A` (the rest being `default`).
283Patterns are the basic building blocks used to define subshifts via forbidden configurations.
284Note that each pattern corresponds to a cylinder, which is the set of configurations
285which agree with this pattern on its support. -/
286structure Pattern (A : Type*) (G : Type*) [Inhabited A] where
287 /-- The full configuration in the full shift `A^G`. -/
288 config : G → A
289 /-- Finite support of the pattern. -/
290 support : Finset G
291 /-- Outside the support, `config` takes the default value of `A`. -/
292 condition : ∀ g ∉ support, config g = default

298/-- `p.mulOccursInAt x g` means that the finite pattern
299`p` appears in the configuration `x`
300at position `g`.
301 
302Formally: for every position `h` in the support of `p`, the value of the configuration
303at `g * h` coincides with the value of `p.config` at `h`.
304 
305Intuitively, if you shift the configuration `x` by `g` (using `mulShift g`),
306then on the support of `p` you exactly recover the pattern `p`. This is the basic
307notion of "pattern occurrence" used to define subshifts via forbidden patterns. -/
308@[to_additive Pattern.occursInAt
309/-- `p.occursInAt x g` means that the finite pattern `p` appears in the configuration `x`
310at position `g`.
311 
312Formally: for every position `h` in the support of `p`, the value of the configuration
313at `g + h` coincides with the value of `p.config` at `h`.
314 
315Intuitively, if you shift the configuration `x` by `g` (using `shift g`),
316then on the support of `p` you exactly recover the pattern `p`. This is the basic
317notion of "pattern occurrence" used to define subshifts via forbidden patterns. -/]
318def Pattern.mulOccursInAt (p : Pattern A G) (x : G → A) (g : G) : Prop :=
319 ∀ (h) (_ : h ∈ p.support), x (g * h) = p.config h

321/-- `mulForbidden F` is the set of configurations that avoid every pattern in `F`.
322 
323Formally: `x ∈ mulForbidden F` if and only if for every pattern `p ∈ F` and every
324monoid element `g : G`, the pattern `p` does not occur in `x` at position `g`.
325 
326Intuitively, `mulForbidden F` is the shift space defined by declaring the finite set
327(or family) of patterns `F` to be *forbidden*. A configuration belongs to the subshift if and only
328it avoids all the forbidden patterns. -/
329@[to_additive forbidden
330/-- `forbidden F` is the set of configurations that avoid every pattern in `F`.
331 
332Formally: `x ∈ forbidden F` if and only if for every pattern `p ∈ F` and every
333monoid element `g : G`, the pattern `p` does not occur in `x` at position `g`.

348/-- Translate a finite pattern `p` so that it occurs at the translate `v`, before completing into
349a configuration.
350 
351On input `h : G`, we proceed as follows:
352* if `h` lies in the left-translate of the support, i.e. `h ∈ p.support.image (v * ·)`,
353 choose (noncomputably) `w ∈ p.support` with `v * w = h` and return `p.config w`;
354* otherwise return `default`.
355 
356This definition does not assume left-cancellation; it only *chooses* a preimage.
357Uniqueness (and the usual equations such as `Pattern.mulShift p v (v * w) = p.config w`)
358require a left-cancellation hypothesis and are proved in separate lemmas.
359-/
360@[to_additive
361/-- Translate a finite pattern `p` so that it occurs at the translate `v`, before completing into
362a configuration.
363 
364On input `h : G`, we proceed as follows:
365* if `h` lies in the left-translate of the support, i.e. `h ∈ p.support.image (v + ·)`,
366 choose (noncomputably) `w ∈ p.support` with `v + w = h` and return `p.config w`;
367* otherwise return `default`.
368 
369This definition does not assume left-cancellation; it only *chooses* a preimage.
370Uniqueness (and the usual equations such as `Pattern.shift p v (v + w) = p.config w`)
371require a left-cancellation hypothesis and are proved in separate lemmas.
372-/]
373protected noncomputable def Pattern.mulShift (p : Pattern A G) (v : G) : G → A := by
374 classical
375 intro h
376 if hmem : h ∈ p.support.image (v * ·) then
377 -- package existence of a preimage under (v * ·)
378 let ex : ∃ w, w ∈ p.support ∧ v * w = h := by
379 simpa [Finset.mem_image] using hmem
380 exact p.config (Classical.choose ex)
381 else
382 exact default

419 -- existential used in the branch
420 have ex : ∃ w', w' ∈ p.support ∧ v * w' = v * w := by
421 simpa [Finset.mem_image] using hmem
422 -- open the `if` branch as returned by the definition
423 have h1 : p.mulShift v (v * w) = p.config (Classical.choose ex) := by
424 simp [Pattern.mulShift, hmem]
425 -- the chosen witness equals w by left-cancellation
426 have hwv' : v * Classical.choose ex = v * w := (Classical.choose_spec ex).2
427 have h_eq : Classical.choose ex = w := mul_left_cancel hwv'

525of these closed sets over `p ∈ F` and `v ∈ G`. -/
526@[to_additive isClosed_forbidden /-- Avoiding a fixed family of patterns is a closed
527condition (in the product topology on `G → A`).
528 
529Since each occurrence set `{ x | p.occursInAt x v }` is open (when `A` is discrete),
530its complement `{ x | ¬ p.occursInAt x v }` is closed; `forbidden F` is the intersection
531of these closed sets over `p ∈ F` and `v ∈ G`. -/]
532lemma isClosed_mulForbidden [DiscreteTopology A] (F : Set (Pattern A G)) :
533 IsClosed (mulForbidden F) := by
534 rw [mulForbidden]
535 -- Rewrite as an intersection indexed by `p ∈ F` and `v : G`.
536 have h_eq : {x | ∀ p ∈ F, ∀ v : G, ¬ p.mulOccursInAt x v}
537 = ⋂ (p : Pattern A G) (hp : p ∈ F) (v : G), {x | ¬ p.mulOccursInAt x v} := by ext; simp
538 rw [h_eq]
539 -- Now prove that this big intersection is closed.
540 refine isClosed_iInter (fun p => ?_)
541 refine isClosed_iInter (fun hp => ?_)
542 refine isClosed_iInter (fun v => ?_)
543 -- For each `p, hp, v`, the section is the complement of an open occurrence set.
544 have : {x | ¬ p.mulOccursInAt x v} = {x | p.mulOccursInAt x v}ᶜ := by ext; simp
545 simpa [this, isClosed_compl_iff] using isOpen_mulOccursInAt (A := A) (G := G) p v
546 
547/-- Occurrence sets are closed. -/
548@[to_additive isClosed_occursInAt]
549lemma isClosed_mulOccursInAt [T1Space A] (p : Pattern A G) (g : G) :
550 IsClosed { x | p.mulOccursInAt x g } := by
551 simpa [mulOccursInAt_eq_cylinder] using isClosed_cylinder _ _

585/-- Patterns with support exactly `U` form a finite set. -/
586lemma finite_setOf_pattern_support_eq
587 {A G : Type*} [Finite A] [Inhabited A]
588 (U : Finset G) :
589 ({p : Pattern A G | p.support = U}).Finite := by
590 -- 1. Upgrade Finite A to Fintype A locally
591 cases nonempty_fintype A
592 classical
593 -- Patterns with support U biject with (U → A) via restriction/extension
594 let e : { p : Pattern A G // p.support = U } ≃ (U → A) :=
595 { toFun := fun p i => p.1.config i.1
596 invFun := fun f => ⟨{ config := fun g => if h : g ∈ U then f ⟨g, h⟩ else default,
597 support := U,
598 condition := fun g hg => by simp [hg] }, rfl⟩
599 left_inv := by
600 rintro ⟨⟨cfg, dom, cond⟩, hU⟩
601 simp only at hU; subst hU
602 apply Subtype.ext
603 simp only [Pattern.mk.injEq, and_true]
604 funext g
605 by_cases hg : g ∈ dom
606 · simp [hg]
607 · simp [hg, cond g hg]
608 right_inv := fun f => by ext i; simp [i.2] }
609 let : Fintype { p : Pattern A G | p.support = U } := Fintype.ofEquiv (U → A) e.symm
610 apply toFinite
